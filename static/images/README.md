# Trompo King Images Directory

This directory stores all images for the Trompo King website.

## Image Files

### Required Images

1. **logo.png** (optional)
   - Logo image displayed in navbar and homepage
   - Recommended size: 150x150px (square)
   - Ideal format: PNG with transparent background
   - If not provided, the site displays a colorful text logo with emoji

2. **tacos.jpg**
   - Featured tacos image for gallery
   - Recommended size: 400x300px or larger
   - Displays on homepage gallery section

3. **burritos.jpg**
   - Featured burritos image for gallery
   - Recommended size: 400x300px or larger

4. **enchiladas.jpg**
   - Featured enchiladas image for gallery
   - Recommended size: 400x300px or larger

5. **quesadillas.jpg**
   - Featured quesadillas image for gallery
   - Recommended size: 400x300px or larger

6. **restaurant.jpg**
   - Restaurant interior or storefront image
   - Recommended size: 400x300px or larger

7. **team.jpg**
   - Team/staff photo for gallery
   - Recommended size: 400x300px or larger

## How to Add Images

1. **Prepare your image** - Ensure it's in JPG or PNG format and properly sized
2. **Save to this folder** - Place the image file in this `static/images/` directory
3. **Name it correctly** - Use the exact filename from the list above
4. **No restart needed** - The website will automatically display your new image

## Image Optimization Tips

- **Size**: Compress images to reduce page load time (aim for < 200KB each)
- **Format**: Use JPG for photos, PNG for images with transparency
- **Dimensions**: Maintain 4:3 aspect ratio (e.g., 400x300px, 800x600px)
- **Quality**: High quality but compressed (72-85% JPG quality)

### Free Tools for Image Optimization
- [TinyPNG](https://tinypng.com/) - Easy batch compression
- [JPEGmini](https://www.jpegmini.com/) - For JPG optimization
- [ImageOptim](https://imageoptim.com/) - For Mac
- [Paint.NET](https://www.getpaint.net/) - Free image editor

## Troubleshooting

**Image not showing?**
1. Check the filename matches exactly (case-sensitive on Linux/Mac)
2. Ensure the image is in the `static/images/` folder
3. Refresh the browser (Ctrl+F5 or Cmd+Shift+R)
4. Check browser console for any error messages

**Image looks low quality?**
1. The image may need to be larger
2. Try using a higher quality source image
3. Avoid upscaling - use original size or larger
